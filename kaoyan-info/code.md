# 考研信息收集技能 - 实现代码

## 实施模式（安全机制）

| 模式 | 行为 | 文件修改 | 适用场景 |
|------|------|----------|----------|
| **① 仅收集信息** | 收集并展示信息 | ❌ 不创建任何文件 | 用户只想了解信息 |
| **② 生成文档草稿** | 生成内容预览 | ❌ 不保存文件 | 用户想预览后再决定 |
| **③ 更新现有文档** | 修改已存在文档 | ✅ 需用户确认 | 用户明确要求更新 |

---

## 工作范围（Scope）

| 范围 | 行为 | 用例示例 |
|------|------|----------|
| **单院校** | 只处理一个院校/专业 | "收集湖大控制工程信息" |
| **多院校对比** | 收集多个院校并生成对比 | "对比湖大、中南、湖师大" |
| **全局更新** | 批量更新所有文档 | "更新所有分数线到2025年" |

---

## 信息收集策略

### 时间优先级规则

```
1. 优先获取最新招生年度信息
2. 若最新未发布 → 标注"暂未发布，使用XX年数据"
3. 若使用历史数据 → 必须在数据旁明确标注年份
```

### 来源优先级规则

```
1. 用户明确提供的来源（最高优先级）
2. 官方网站（研招网、院校研究生院官网）
3. 其他可靠来源（需标注来源）
```

### 来源标注规范

- 用户提供的：`来源：用户提供`
- 官方网站：`来源：[官网名称](链接)`
- 其他来源：`来源：[来源名称](链接) ⚠️请核实官方信息`

## 工作流程

### 0. 准备阶段

#### 确认执行模式
```
请选择执行模式：
① 仅收集信息 - 不创建任何文件
② 生成文档草稿 - 生成内容预览，不保存
③ 更新现有文档 - 修改已存在的文档
```

#### 确认工作范围
```
请选择工作范围：
- 单院校（如：湖大控制工程）
- 多院校对比（如：湖大、中南、湖师大）
- 全局更新（如：更新所有分数线）
```

### 1. 信息收集流程

1. 明确需求：确认目标院校、专业、年份
2. 搜索官方信息
3. 提取关键信息
4. 验证信息时效性
5. 记录数据来源

### 2. 文档创建/更新流程

- 模式②：仅展示预览，不保存
- 模式③：保存到用户配置的目录，需确认
