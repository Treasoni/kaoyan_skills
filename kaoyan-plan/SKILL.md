# 考研规划 Skill

本技能是日常学习调度入口，负责把当前进度、可用时间、疲劳度和任务欠账转成可执行的每日/每周计划；当用户汇报完成情况时，负责更新每日计划、完成记录和各科进度文件。

## 先判定任务类型

| 用户意图 | 处理动作 | 必读模块 |
| --- | --- | --- |
| 今天怎么学、安排计划、生成课表 | 生成每日/每周学习计划 | 本文件 + `features.md` |
| 昨天/今天完成了什么、帮我记录完成 | 更新完成记录和进度文件 | 本文件 + `completion-report.md` |
| 欠账太多、补计划、没完成 | 进入欠账评估和恢复模式 | 本文件 + `features.md` |
| 周日复盘、本周总结 | 生成周复盘和下周调整 | 本文件 + `templates.md` |
| 跨科协调、权重调整 | 调整全科优先级和调度信号 | 本文件 + `dispatch.md` |
| 需要算法/API细节 | 查看实现入口和脚本索引 | `code.md` |

## 输入模式

| 模式 | 用户需要提供 | 处理方式 |
| --- | --- | --- |
| 极简模式 | 课表/可用时间 + 今日偏好，可缺省 | 优先读取现有计划和进度，少追问 |
| 标准模式 | 极简信息 + 科目偏重/考试节点 | 调整科目权重和阶段目标 |
| 高级模式 | 进度、学习时长、疲劳度、欠账 | 做完整重排、补账和降级 |

默认使用极简模式。只有会覆盖个人内容、长期规则或关键事实缺失会导致明显错误时才追问。

## 计划生成规则

### 科目权重

默认优先级：数学 > 英语 > 专业课 > 政治。

默认权重：数学 1.5，英语 1.2，专业课 1.0，政治 0.6。距离考试越近，专业课和政治权重逐步提高；具体阶段权重见 `dispatch.md`。

### 时段偏好

| 时段 | 优先安排 |
| --- | --- |
| 08:00-12:00 | 数学、英语单词 |
| 14:00-18:00 | 英语阅读、专业课 |
| 19:00-23:00 | 专业课、政治复盘 |
| 23:00 后 | 仅复习，不排新内容 |

### 最小时间块

| 任务类型 | 最小时长 |
| --- | --- |
| 数学中高难度 | 1.5 小时 |
| 英语阅读、专业课、数学概念 | 1.0 小时 |
| 单词、政治选择题 | 15-20 分钟，可碎片化 |

不满足最小时长时，替换为单词、政治选择题、错题回看或轻复盘。

### 疲劳度降级

疲劳度 = 主观感受 0.6 + 行为数据 0.4。

| 疲劳度 | 策略 |
| --- | --- |
| < 0.3 | 正常排期，休息约 15% |
| 0.3-0.5 | 正常排期，休息约 20% |
| 0.6-0.8 | 削减 10%-20% 任务量，休息约 25% |
| > 0.8 | 削减 30% 以上，开启长休息或休息日 |

连续 3 天疲惫时，进入心理调节模式：降低新内容比例，优先复习、整理和可完成的小任务。

## 欠账处理

| 欠账时长 | 等级 | 处理策略 |
| --- | --- | --- |
| < 1 小时 | 轻微 | 用碎片时间补完 |
| 1-3 小时 | 中等 | 压缩低优先级任务补课 |
| 3-10 小时 | 严重 | 建议半天或整块补课 |
| > 10 小时 | 熔断 | 暂停新内容，进入恢复模式 |

恢复模式下不要继续堆新计划，要先把欠账拆成可关闭的小块。

## 输出格式

生成计划时，输出应包含：

1. 今日总策略：一句话说明主线和降级/补账状态。
2. 时间表：按时间块列出任务、科目、目标产出和休息。
3. 兜底方案：如果晚起、疲劳或被打断，保留哪些任务。
4. 检查点：当天结束时用户该如何汇报完成情况。

更新完成情况时，必须先读取 `completion-report.md`，按清单逐项更新，不得只改每日计划。

## 数据保护

严禁覆盖或删除以下内容，只能追加或跳过：

- `## 个人笔记` / `## My Insights`
- `## 随手记` / `## Quick Notes`
- `## 学习心得` / `## Learnings`
- `## 踩坑记录` / `## Pitfalls`
- `## 待探索` / `## TODO`
- 任何以 `> [!personal]` 开头的 callout 块

写入含表格的 Markdown 后，必须检查 Obsidian 表格格式；表格内不要使用会破坏渲染的 LaTeX 管道符。
