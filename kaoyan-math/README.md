🧠 Kaoyan Math System

考研数学结构化学习与迭代笔记系统

📌 项目定位

Kaoyan Math System 是一个基于 Claude Code 构建的 考研数学学习系统化方案。

本项目的目标不是简单生成笔记，而是：

构建一个可持续迭代的数学学习系统
让“理解 → 反馈 → 修正 → 强化”成为循环

适用于：

考研数学（数一 / 数二 / 数三）

使用 Obsidian 进行知识管理

希望系统化学习，而不是零散做题的学习者

🎯 核心理念

传统学习流程：

听课 → 做题 → 错 → 再错

本系统学习流程：

原始笔记
   ↓
结构化生成
   ↓
学习理解
   ↓
问题反馈
   ↓
笔记更新
   ↓
错误模式沉淀
   ↓
强化训练


它不是一次性整理工具，而是：

一个可迭代的数学认知系统

🏗 系统结构
1️⃣ 三大模块覆盖

高等数学

线性代数

概率论与数理统计

完整知识树已内置于 Skill 设计中。

2️⃣ 单知识点文件结构

每个知识点独立成文件，包含：

- 原始定义
- 直观理解 💡
- 定理条件 ⚠️
- 考试重点 ⭐
- 典型题型
- 解题方法
- 易错点 ⚠️
- 例题
- 我的理解记录 🧠
- 我的错误类型 📌
- 补充说明（带 UPDATE 标记）

3️⃣ 个性化追踪系统
🧠 我的理解记录

记录：

初始理解

误区来源

学习转折点

未解决问题

📌 我的错误类型

分类管理：

条件遗漏

方法误选

推导跳步

计算失误

个人思维惯性

目的：

降低重复犯错概率

🧩 技术特点

Markdown / Obsidian 友好

支持知识点分文件管理

支持更新标记（UPDATE）

支持错误模式追踪

支持考试类型区分（数一/数二/数三）

支持持续迭代

⚠ 使用边界

本系统：

✅ 提供结构
✅ 提供整理
✅ 提供反馈修正

但不替代：

做题量

推导训练

计算熟练度

真题反复刷

系统是加速器
不是发动机
