# Knowledge Base Organizer

> 智能整理你的 Obsidian 知识库

一个用于组织 Obsidian Markdown 知识库的技能，通过文档链接、MOC 创建、概念提取和知识缺口分析，帮助你构建结构化的知识网络。

## 功能特性

### 文档链接
- 分析文档内容，识别主题相关性
- 在文档之间创建 wikilink 关联
- 智能推荐"潜在关联"

### MOC 文档管理
- 创建按主题分类的 MOC（如：MOC - 前端开发、MOC - Python）
- 创建按文档类型的 MOC（如：概念笔记、教程笔记、项目笔记）
- 自动更新现有 MOC

### 概念抽取
- 提取核心概念和关键词
- 生成 tags 建议
- 标准化术语

### 文档类型识别
识别并分类以下类型：
- 📝 **概念笔记** - 定义、原理、术语解释
- 📚 **教程笔记** - 步骤、方法、学习路径
- 🛠️ **项目笔记** - 项目记录、代码片段
- ❓ **问题解决记录** - bug 修复、troubleshooting

### 自动生成摘要
- TL;DR（一句话总结）
- 关键要点列表
- 使用场景标签

### 反向链接建议
- 已有关联分析
- 潜在关联推荐（同技术栈、同问题链路）

### 知识缺口提示
- 检测主题下的子主题缺失
- 标记相关但未链接的孤立文档

### 复习/行动提取
- 提取 TODO 项
- 标记可复用代码
- 生成复习清单

## 使用方式

### 触发关键词
- "整理文档"、"整理笔记"
- "生成 MOC"、"更新 MOC"
- "分析知识库"、"扫描文档"
- "建立文档关联"、"添加链接"
- "提取概念"、"生成摘要"
- "检查知识缺口"

### 工作模式

| 模式 | 行为 | 文件修改 |
|------|------|----------|
| **只读模式** | 分析 → 报告结果 | 不修改任何文件 |
| **建议+确认模式** | 分析 → 展示建议 → 用户确认 → 执行 | 修改文件（需确认） |

## 配置选项

| 参数 | 说明 | 默认值 |
|------|------|--------|
| scope | 扫描范围: single-note, folder, vault | single-note |
| max_links | 单次最大链接建议数 | 10 |
| relevance_threshold | 相关性阈值 (0-100) | 60 |

## MOC 文件组织结构

```
.moc/
├── MOC - 按主题/
│   ├── MOC - 前端开发.md
│   ├── MOC - 后端开发.md
│   ├── MOC - 数据库.md
│   └── ...
└── MOC - 按类型/
    ├── MOC - 概念笔记.md
    ├── MOC - 教程笔记.md
    ├── MOC - 项目笔记.md
    └── MOC - 问题记录.md
```

## 相关性评分

- 直接主题匹配: 100分
- 同技术栈/同领域: 80分
- 术语引用关系: 60分
- 弱关联: 40分

只显示评分 ≥ 60 的建议

## 工作流

1. **理解请求** - 识别用户意图，确定文档范围，选择实施模式
2. **扫描分析** - 读取目标文档，提取内容特征
3. **生成建议** - 生成文档链接、MOC 更新、概念抽取结果
4. **执行** - 展示建议，等待用户确认，执行修改操作
