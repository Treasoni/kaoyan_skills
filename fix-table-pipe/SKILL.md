# 修复表格管道符问题

## 问题背景

在 Markdown 表格中，LaTeX 公式的绝对值符号 `|` 会被误解析为表格列分隔符，导致表格结构破坏。

**错误示例**：
```markdown
| $|f(x)|$ 连续 | $f(x)$ 不连续 |
```
会被解析为 4 列而不是 2 列。

## 常见表格格式问题

### 1. 表格前缺少空行

> ⚠️ **重要**：表格前必须有空行才能正确渲染！

```markdown
# ❌ 错误 - 表格不会渲染
**表格标题**：
| 列1 | 列2 |
|-----|-----|
| A   | B   |

# ✅ 正确 - 表格会正确渲染
**表格标题**：

| 列1 | 列2 |
|-----|-----|
| A   | B   |
```

### 2. LaTeX 绝对值管道符冲突

## 解决方案

将表格内的 `|...|` 转换为 `\vert...\vert`：

```markdown
| $\vert f(x) \vert$ 连续 | $f(x)$ 不连续 |
```

## 处理规则

### 1. 表格检测

Markdown 表格的特征是包含分隔行：
```
| 列1 | 列2 |
|-----|------|
| 内容 | 内容 |
```

只处理分隔行之后的表格内容行。

### 2. 转换模式

在表格内容行中，转换以下模式：

| 原始格式 | 转换后 |
|---------|--------|
| `$|expr|$` | `$\vert expr \vert$` |
| `$$|expr|$$` | `$$\vert expr \vert$$` |
| `$|expr| + |expr|$` | `$\vert expr \vert + \vert expr \vert$` |

### 3. 不处理的情况

- **Callout 块内**：`> [!tip] ... $|f(x)|$ ...` 不需要转换
- **已使用 \vert**：`$\vert f(x) \vert$` 已是正确格式
- **已使用 \left\right**：`$\left| f(x) \right|$` 已是正确格式
- **代码块内**：不处理

### 4. 保留格式

- 保持表格对齐方式（`:---:`, `:---`, `---:`）
- 保持原有缩进和空格

## 工作流程

1. **读取文件**：读取用户指定的 Markdown 文件
2. **定位表格**：找到所有表格的起始和结束位置
3. **逐行处理**：只处理表格内的内容行
4. **应用转换**：将 `$|...|$` 转换为 `$\vert...\vert$`
5. **写回文件**：使用 Edit 工具更新文件

## 示例

**输入**：
```markdown
| $f(x)$ 连续 | 对应的 $|f(x)|$ 图像 |
|:---:|:---:|
| ![](a.png) | ![](d.png) |
```

**输出**：
```markdown
| $f(x)$ 连续 | 对应的 $\vert f(x) \vert$ 图像 |
|:---:|:---:|
| ![](a.png) | ![](d.png) |
```
